# 宝塔面板部署指南 — 后端

> 本篇介绍如何用**宝塔面板(BT Panel)**将 WowBlog 后端(FastAPI)部署到生产服务器。
> 前端部署请参阅 [`frontend/docs/宝塔面板部署.md`](../../frontend/docs/宝塔面板部署.md)。

提供两种方案,任选其一:

- **方案 A:Docker 部署(推荐)** — 项目自带 `docker-compose.yml`,MySQL/Redis/后端一键拉起,最省心、最不易出错。
- **方案 B:Python 项目管理器(传统)** — 用宝塔的 Python 项目管理器跑 FastAPI,MySQL/Redis 用宝塔组件。适合不想用 Docker、或资源紧张的小服务器。

> **版本兼容**:项目最低支持 **Python 3.12** 与 **MySQL 5.7**。宝塔面板常见的 3.12.0 / 5.7 可直接部署(走方案 B)。代码未使用任何 3.13 / 8.0 独有特性。详见文末[《MySQL 5.7 / Python 3.12 部署注意》](#mysql-57--python-312-部署注意)。

---

## 方案 A:Docker 部署(推荐)

### A.0 前置
宝塔面板 → **软件商店** → 搜索安装 **Docker 管理器**(含 Docker 与 compose)。

### A.1 上传后端代码
把仓库的 `backend/` 目录上传到服务器,例如 `/www/wwwroot/wowblog-backend/`。

```bash
# 终端里确认目录结构(应有 docker-compose.yml、Dockerfile、requirements.txt、app/)
ls /www/wwwroot/wowblog-backend
```

### A.2 配置环境变量
```bash
cd /www/wwwroot/wowblog-backend
cp .env.example .env
```
编辑 `.env`,**至少**设置以下项(其余按需):

```ini
# 数据库(MySQL 容器初始化用)
MYSQL_ROOT_PASSWORD=<强随机密码>
MYSQL_DATABASE=wowblog
MYSQL_USER=wowblog
MYSQL_PASSWORD=<强随机密码>

# 后端连接数据库(compose 会把 DATABASE_HOST 注入为 mysql 容器)
DATABASE_HOST=mysql
DATABASE_PORT=3306
DATABASE_NAME=wowblog
DATABASE_USER=wowblog
DATABASE_PASSWORD=<同上 MYSQL_PASSWORD>

# JWT 密钥(必须,生产用强随机)
JWT_SECRET_KEY=<下面命令生成的值>
ACCESS_TOKEN_EXPIRE_MINUTES=30

# 环境
ENVIRONMENT=production
DEBUG=false

# CORS(填你的前端域名)
ALLOWED_ORIGINS=https://你的域名

# Redis(compose 注入 REDIS_HOST=redis)
REDIS_HOST=redis
REDIS_PORT=6379
```

生成 JWT 密钥:
```bash
python -c "import secrets; print(secrets.token_urlsafe(32))"
# 或
openssl rand -base64 32
```

### A.3 构建并启动
```bash
cd /www/wwwroot/wowblog-backend
docker compose up -d --build
```
首次构建会拉取 `python:3.13-slim`、`mysql:8.0`、`redis:7-alpine` 镜像并安装依赖,耗时数分钟。

查看状态:
```bash
docker compose ps            # 四个容器应都是 running / healthy
docker compose logs -f backend   # 看后端日志
curl http://127.0.0.1:8000/health   # 应返回 {"status":"ok"}
```

### A.4 初始化数据库表
项目当前 Alembic 只有 baseline,业务表结构以 `migrations/*.sql` 为准。在 MySQL 容器里执行:

```bash
# 查看迁移文件
ls /www/wwwroot/wowblog-backend/migrations/

# 依次执行(按文件名顺序)——把每个 *.sql 灌进 mysql 容器
for f in /www/wwwroot/wowblog-backend/migrations/*.sql; do
  echo "执行 $f ..."
  docker exec -i wowblog_mysql mysql -uroot -p"$MYSQL_ROOT_PASSWORD" wowblog < "$f"
done

# 也执行根目录的隐私字段迁移(如有)
docker exec -i wowblog_mysql mysql -uroot -p"$MYSQL_ROOT_PASSWORD" wowblog \
  < /www/wwwroot/wowblog-backend/add_privacy_fields.sql
```

> 也可在宝塔 → **数据库** → 对应容器/实例 → phpMyAdmin 里手动导入这些 SQL。

### A.5 Web 反代(前端站点)
后端容器映射的是宿主机 `8000` 端口,生产**不要**让 8000 直接对外,而是通过前端站点的反代 `/api`(Nginx 见前端文档 4.1,Apache 见 4.2)。

如需让 8000 仅本机访问,把 `docker-compose.yml` 里 backend 的 `ports: - "8000:8000"` 改为 `expose: - "8000"`(仅容器网络可达,配合 Nginx 反代 `http://127.0.0.1:8000` 仍可用,因为它走宿主机 loopback)。

### A.6 常用运维命令
```bash
docker compose restart backend     # 重启后端
docker compose logs --tail=200 backend
docker compose down                 # 停止全部
docker compose up -d --build        # 代码更新后重建后端
```

## 方案 B:Python 项目管理器(传统)

适合不用 Docker 的场景。需手动管理 MySQL、Redis、Python 进程。

### B.0 安装软件(软件商店)
- **MySQL 5.7 / 8.0**(项目两者都支持,宝塔 5.7 可用)
- **Redis**
- **Python 项目管理器**(宝塔插件)

### B.1 创建数据库
宝塔 → **数据库** → **添加数据库**:
- 数据库名:`wowblog`
- 用户名:`wowblog`
- 密码:强随机
- 字符集:`utf8mb4` / `utf8mb4_unicode_ci`(**MySQL 5.7 必须显式选,默认是 latin1**)

记录端口(宝塔 MySQL 默认 3306,若改过以实际为准)。

导入表结构:进入该库的 **phpMyAdmin** → 导入,依次把 `backend/migrations/*.sql` 与 `backend/add_privacy_fields.sql` 导入。

### B.2 上传代码并添加 Python 项目
1. 上传 `backend/` 到 `/www/wwwroot/wowblog-backend/`
2. 宝塔 → **Python 项目管理器** → **添加项目**:
   - 项目路径:`/www/wwwroot/wowblog-backend`
   - Python 版本:**3.12**(项目最低要求 3.12;宝塔 3.12.0 可用,3.13 亦可)
   - 框架:选 **FastAPI**(或通用)
   - 启动方式:**命令行 / uvicorn**
   - 安装依赖:在项目虚拟环境里执行
     ```bash
     pip install -r requirements.txt
     ```
     (国内服务器若 pip 慢,加镜像:
     `pip install -i https://pypi.tuna.tsinghua.edu.cn/simple -r requirements.txt`)
   - 启动命令:
     ```bash
     uvicorn app.main:app --host 127.0.0.1 --port 8000 --workers 4
     ```
     > 多 worker 生产推荐用 gunicorn:
     > `gunicorn app.main:app -w 4 -k uvicorn.workers.UvicornWorker -b 127.0.0.1:8000`

### B.3 配置环境变量
```bash
cd /www/wwwroot/wowblog-backend
cp .env.example .env
```
编辑 `.env`(注意:这里 MySQL/Redis 都是宝塔本机组件,不是容器):
```ini
DATABASE_HOST=127.0.0.1
DATABASE_PORT=3306                # 宝塔 MySQL 端口
DATABASE_NAME=wowblog
DATABASE_USER=wowblog
DATABASE_PASSWORD=<宝塔数据库密码>
JWT_SECRET_KEY=<生成的强随机>
ACCESS_TOKEN_EXPIRE_MINUTES=30
ENVIRONMENT=production
DEBUG=false
ALLOWED_ORIGINS=https://你的域名
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
```

### B.4 启动并验证
在 Python 项目管理器里**启动**项目,然后:
```bash
curl http://127.0.0.1:8000/health   # 应返回 {"status":"ok"}
```

### B.5 Web 反代
同方案 A 的 A.5:用前端站点的反代 `/api` → `127.0.0.1:8000`(Nginx 见前端文档 4.1,Apache 见 4.2)。

## MySQL 5.7 / Python 3.12 部署注意

宝塔若只提供 **MySQL 5.7** 和 **Python 3.12**(而非 Docker 方案的 8.0 / 3.13),完全可以部署 —— 项目代码未使用任何 8.0 / 3.13 独有特性(无 CHECK 约束、生成列、窗口函数、CTE、`utf8mb4_0900_*`;无 `@deprecated`、`ReadOnly` 等 3.13-only API)。走**方案 B** 即可,注意三点:

### 1. 数据库字符集(最重要)
MySQL 5.7 默认字符集是 `latin1`(**8.0 才默认 `utf8mb4`**)。**建库时必须显式指定 utf8mb4**,否则中文乱码:
```sql
CREATE DATABASE wowblog CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```
项目的表 DDL 已写 `CHARSET=utf8mb4`,但库级默认仍要设对 —— 宝塔「添加数据库」时字符集选 `utf8mb4`,或建库后执行上述 SQL。

### 2. 认证插件
MySQL 5.7 默认认证插件 `mysql_native_password`,aiomysql / pymysql 原生支持,**无需任何处理**(比连 8.0 默认的 `caching_sha2_password` 更省心)。`.env` 的 `DATABASE_USER` 用 root 或自建用户均可。

### 3. Python 3.12 运行
项目 `pyproject.toml` 声明 `requires-python = ">=3.12"`,3.12.0 满足。`pip install -r requirements.txt` + `uvicorn app.main:app` 直接可跑,不要求 3.13。

> **Docker 方案(方案 A)** 默认仍用 `python:3.13-slim` 与 `mysql:8.0` 镜像(较新、更稳定)。若你的宝塔支持 Docker、却想固定用 5.7/3.12,可自行把 `Dockerfile` 基础镜像改为 `python:3.12-slim`、`docker-compose.yml` 的 `mysql:8.0` 改为 `mysql:5.7`。

## 环境变量速查

| 变量 | 必填 | 说明 |
|---|---|---|
| `DATABASE_PASSWORD` | ✅ | 数据库密码(`database.py` 在启动时强制检查) |
| `JWT_SECRET_KEY` | ✅(生产) | JWT 签名密钥;生产缺失会拒绝启动 |
| `DATABASE_HOST` | ✅ | Docker 方案=`mysql`;传统方案=`127.0.0.1` |
| `MYSQL_ROOT_PASSWORD` / `MYSQL_USER` / `MYSQL_PASSWORD` / `MYSQL_DATABASE` | Docker 方案 | MySQL 容器初始化用 |
| `REDIS_HOST` | 推荐 | Docker=`redis`;传统=`127.0.0.1` |
| `ENVIRONMENT` | 推荐 | `production` 时关闭 `/docs`、强制 `JWT_SECRET_KEY` |
| `ALLOWED_ORIGINS` | ✅ | CORS 允许的前端域名,逗号分隔 |
| `DEFAULT_MEMBER_PASSWORD` | 可选 | 后台新建成员默认密码 |
| `MAX_UPLOAD_SIZE` | 可选 | 上传大小上限(MB),默认 10 |

## 数据库迁移说明

- 项目使用 **Alembic**,但目前只含 baseline(`alembic/versions/` 里只有初始化版本)。
- **业务表结构以 `migrations/*.sql` 为准**,首次部署必须执行这些 SQL(见 A.4 / B.1)。
- 之后 schema 变更应逐步纳入 Alembic:
  ```bash
  alembic upgrade head
  ```

## 常见问题

### 启动报 `DATABASE_PASSWORD environment variable not set`
`.env` 没配 `DATABASE_PASSWORD`,或 `.env` 没被加载(Docker 方案 compose 自动注入;传统方案需在 Python 项目的工作目录有 `.env`)。

### 启动报 `JWT_SECRET_KEY ... not set in production`
生产环境(`ENVIRONMENT=production`)必须设置 `JWT_SECRET_KEY`。生成见上文。

### 数据库连接被拒(`Access denied` / `Can't connect`)
- Docker 方案:确认 `DATABASE_PASSWORD` 与 `MYSQL_PASSWORD` 一致,且 MySQL 容器健康(`docker compose ps`)。
- 传统方案:确认数据库用户授权了 `wowblog` 库、端口正确、`127.0.0.1` 可连。

### 接口 502
后端进程没起或端口不对。检查 `8000` 监听:`ss -tlnp | grep 8000`,核对 Nginx `proxy_pass`。

### `/api/docs` 打不开
生产环境(`ENVIRONMENT=production`)已主动关闭 `/docs`、`/redoc`、`/openapi.json`。开发环境设 `ENVIRONMENT=development` 可恢复。

### Redis 连不上(限流不生效)
确认 Redis 运行中:`redis-cli ping` 应返回 `PONG`(传统方案);Docker 方案确认 `redis` 容器健康。

### 中文乱码(MySQL 5.7)
库的字符集不是 utf8mb4。执行 `ALTER DATABASE wowblog CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;`,并确认连接串带 `charset`(项目已配 `utf8mb4`)。

## 安全清单(上线前)

- [ ] `ENVIRONMENT=production`、`DEBUG=false`
- [ ] `JWT_SECRET_KEY` 为强随机值,与开发环境不同
- [ ] 数据库密码强随机,`MYSQL_ROOT_PASSWORD` 仅服务器内部使用
- [ ] `ALLOWED_ORIGINS` 仅含你的真实前端域名
- [ ] 后端 `8000` 端口**不对公网开放**,仅通过 Nginx/Apache 反代
- [ ] 已配置 HTTPS(前端站点 → SSL → Let's Encrypt)
- [ ] 宝塔防火墙仅放行 22/80/443(及面板端口)
- [ ] 已设置数据库定期备份(宝塔 → 计划任务 → mysqldump)
