# 宝塔面板部署指南 — 前端

> 本篇介绍如何用**宝塔面板(BT Panel)**将 WowBlog 前端(Vue 3)部署到生产服务器。
> 后端部署请参阅 [`backend/docs/宝塔面板部署.md`](../../backend/docs/宝塔面板部署.md)。

## 部署架构

```
浏览器 ──HTTPS──▶ Nginx(宝塔站点)
                    ├── /            → 托管前端 dist 静态文件(SPA)
                    ├── /api/        → 反代后端 FastAPI (127.0.0.1:8000)
                    └── /ws、/api/ws → WebSocket 反代后端
```

前端构建为纯静态文件(dist/),由宝塔的 Nginx 托管;API 与 WebSocket 通过 Nginx 反向代理到后端服务,实现**同域部署**(无跨域、无需为前端单独配 CORS)。

---

## 前置条件

- 已安装**宝塔面板**(推荐 7.x+,含 LNMP,即 Nginx)
- **后端已部署**并监听 `127.0.0.1:8000`(见后端文档)
- 域名已解析到本服务器(或先用 IP 测试)
- 本地或服务器有 **Node.js ≥ 20**(用于构建;若服务器无 Node,可在本地 build 后只上传 dist)

---

## 1. 构建前端

在本地(或服务器)项目 `frontend/` 目录执行:

```bash
cd frontend
npm install

# 配置生产环境接口地址(同域部署用相对路径 /api 最简单)
echo 'VITE_API_BASE_URL=/api' > .env.production
# 如前后端不同域,改为绝对地址,如:
# echo 'VITE_API_BASE_URL=https://api.yourdomain.com/api' > .env.production

# 视情况调整其他变量
cp .env.example .env.local   # 参考模板,按需修改

npm run build                # 产出 dist/
```

构建产物在 `frontend/dist/`。`npm run build` 会自动剔除 `console.*`。

> 若服务器无 Node,在本地完成 build,只把 `dist/` 上传到服务器即可。

---

## 2. 宝塔添加站点

1. 宝塔面板 → **网站** → **添加站点**
2. **域名**:填你的域名(如 `www.yourdomain.com`),或用 IP
3. **根目录**:默认 `/www/wwwroot/你的域名`(记住这个路径,后面要用)
4. **PHP 版本**:选**纯静态**(本站点只托管静态文件,不需要 PHP)
5. **数据库**:不创建
6. 点 **提交**

---

## 3. 上传 dist

1. 进入站点根目录 `/www/wwwroot/你的域名/`,**清空**默认生成的 `index.html`(宝塔占位页)
2. 把本地 `frontend/dist/` 里的**所有文件**(含 `index.html`、`assets/`、`favicon.ico` 等)上传到该目录

最终目录结构:

```
/www/wwwroot/你的域名/
├── index.html
├── assets/
│   ├── index-xxx.js
│   ├── index-xxx.css
│   └── ...
└── favicon.ico
```

> 可用宝塔「文件」管理器打包 zip 上传后解压,或用 scp / rsync。

---

## 4. 配置 Nginx(核心)

**网站 → 对应站点 → 设置 → 配置文件**,在 `server { ... }` 块内修改/确认如下配置:

```nginx
server {
    listen 80;
    listen 443 ssl http2;          # 申请 SSL 后启用(见第 5 步)
    server_name yourdomain.com;

    # SSL 证书(申请后宝塔会自动填,这里仅示意)
    # ssl_certificate     /www/server/panel/vhost/cert/你的域名/fullchain.pem;
    # ssl_certificate_key /www/server/panel/vhost/cert/你的域名/privkey.pem;

    root /www/wwwroot/你的域名;    # 指向 dist 所在目录
    index index.html;

    # 上传文件大小限制(与后端 MAX_UPLOAD_SIZE 对齐)
    client_max_body_size 10m;

    # gzip 压缩(减小 JS/CSS 传输体积)
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;
    gzip_min_length 1k;

    # —— 前端 SPA:所有未命中静态文件的请求回退到 index.html ——
    location / {
        try_files $uri $uri/ /index.html;
    }

    # —— API 反向代理到后端 ——
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # WebSocket 支持(后端 /api/ws 走这里)
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";

        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # —— WebSocket 专用入口(前端若用 /ws 连接,则重写到 /api/ws)——
    location /ws {
        rewrite ^/ws /api/ws break;
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_read_timeout 3600s;   # 长连接
    }

    # —— 后端上传的静态资源(头像、图片等),可选:直接由后端提供,或在此反代 ——
    location /static/ {
        proxy_pass http://127.0.0.1:8000;
    }

    # 静态资源长缓存(hashed 文件名,可安全 immutable)
    location ~* \.(?:js|css|woff2?|png|jpg|jpeg|gif|svg|ico)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
        access_log off;
    }
}
```

保存后,宝塔会自动 `nginx -t` 校验。若手动改,记得回面板点「重载配置」或在终端:

```bash
nginx -t && nginx -s reload
```

> **说明**:`/api/` 反代已覆盖 `/api/ws`,WebSocket 正常工作。额外的 `/ws` 规则是为兼容前端用 `/ws` 连接的场景(与开发环境 Vite proxy 行为一致)。

---

## 5. 配置 SSL(强制 HTTPS)

1. 站点 → **SSL** → **Let's Encrypt**
2. 勾选域名 → **申请**(需域名已解析且 80 端口可访问)
3. 申请成功后,开启**强制 HTTPS**

宝塔会自动把证书路径写进 Nginx 配置,并自动续期。

---

## 6. 验证

| 检查项 | 方法 |
|---|---|
| 前端首页 | 浏览器访问 `https://你的域名`,应看到首页 |
| SPA 路由 | 访问某个前端路由(如 `/article/1`)后刷新,不报 404 |
| API 连通 | 尝试登录或打开需要接口的页面,Network 里 `/api/...` 返回 200 |
| WebSocket | 触发通知功能(评论、点赞),实时推送正常 |
| 接口文档 | `https://你的域名/api/docs`(仅开发环境 `ENVIRONMENT=development` 时可用) |

## 故障排查

### 刷新子路由 404
Nginx 缺少 SPA 回退。确认 `location /` 里有 `try_files $uri $uri/ /index.html;`。

### 接口 502 Bad Gateway
后端没起或端口不对。检查后端是否监听 `127.0.0.1:8000`:
```bash
ss -tlnp | grep 8000
```
若后端用了不同端口,同步修改 Nginx 的 `proxy_pass`。

### 接口 404 / 跨域
确认 `VITE_API_BASE_URL` 与 Nginx 反代路径一致。同域部署应设为 `/api`,让请求走 Nginx 的 `/api/` 反代。后端 `.env` 的 `ALLOWED_ORIGINS` 也要包含你的域名。

### WebSocket 连不上
检查 Nginx 的 `/api/` 或 `/ws` location 是否含 `Upgrade` / `Connection upgrade` 三行。`proxy_read_timeout` 建议调大(长连接)。

### 上传文件 413
Nginx 的 `client_max_body_size` 太小,调到 ≥ 后端 `MAX_UPLOAD_SIZE`(默认 10m)。

## 更新前端

重新构建并替换 dist 即可,无需重启 Nginx:

```bash
# 本地
npm run build
# 上传 dist/ 覆盖 /www/wwwroot/你的域名/
```

> 提示: hashed 文件名会让浏览器自动加载新版本;`index.html` 不缓存以保证即时更新。
