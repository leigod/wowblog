# 前端用户登录方案

## 一、整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                      前端登录页面                             │
│  ┌──────────────────────────────────────────────────────┐   │
│  │              - 邮箱/用户名 + 密码登录                  │   │
│  │              - 社交账号登录按钮组                      │   │
│  │              - 手机号验证码登录 (可选)                  │   │
│  └──────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                    后端 OAuth 认证层                          │
│  ┌──────────────┬──────────────┬──────────────────────────┐ │
│  │   Google     │    GitHub    │   Apple Sign In          │ │
│  │   (OAuth)    │   (OAuth)    │   (OAuth + JWT)          │ │
│  ├──────────────┼──────────────┼──────────────────────────┤ │
│  │    微信      │      QQ      │      支付宝               │ │
│  │  (微信开放平台) │ (QQ互联)    │   (支付宝授权)            │ │
│  └──────────────┴──────────────┴──────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                   用户关联与JWT Token                         │
│  • 首次登录：创建新用户账号，绑定OAuth身份                    │
│  • 已登录：直接返回JWT Token                                 │
│  • 支持同一用户绑定多个社交账号                               │
└─────────────────────────────────────────────────────────────┘
```

## 二、数据库设计

需要新建一张 OAuth 绑定表：

```sql
CREATE TABLE wb_user_oauth (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT NOT NULL,                    -- 关联用户ID
    provider VARCHAR(50) NOT NULL,           -- 提供商 (google, github, apple, wechat, qq, alipay)
    provider_user_id VARCHAR(255) NOT NULL,  -- 第三方用户ID
    access_token TEXT,                       -- 访问令牌
    refresh_token TEXT,                      -- 刷新令牌
    email VARCHAR(255),                      -- 第三方邮箱
    avatar_url VARCHAR(500),                 -- 第三方头像
    raw_data JSON,                           -- 原始用户数据
    createtime INT DEFAULT 0,
    FOREIGN KEY (user_id) REFERENCES wb_user(id) ON DELETE CASCADE,
    UNIQUE KEY uk_provider_user (provider, provider_user_id)
);
```

## 三、后端实现方案

### 3.1 依赖安装
```bash
pip install authlib python-multipart httpx
```

### 3.2 配置文件
```python
# app/config/oauth.py
class OAuthConfig:
    # Google
    GOOGLE_CLIENT_ID = "your-google-client-id"
    GOOGLE_CLIENT_SECRET = "your-google-secret"

    # GitHub
    GITHUB_CLIENT_ID = "your-github-client-id"
    GITHUB_CLIENT_SECRET = "your-github-secret"

    # Apple
    APPLE_CLIENT_ID = "your-apple-client-id"
    APPLE_TEAM_ID = "your-team-id"
    APPLE_KEY_ID = "your-key-id"

    # 微信
    WECHAT_APP_ID = "your-wechat-appid"
    WECHAT_APP_SECRET = "your-wechat-secret"

    # QQ
    QQ_CLIENT_ID = "your-qq-client-id"
    QQ_CLIENT_SECRET = "your-qq-secret"

    # 支付宝
    ALIPAY_APP_ID = "your-alipay-appid"
```

### 3.3 API 端点设计
```
POST /api/auth/register          - 邮箱注册
POST /api/auth/login             - 邮箱/用户名登录
GET  /api/auth/oauth/{provider}  - OAuth授权跳转
GET  /api/auth/callback/{provider}- OAuth回调处理
POST /api/auth/refresh           - 刷新Token
POST /api/auth/logout            - 登出
GET  /api/auth/me                - 获取当前用户信息
```

## 五、各平台对接要点

| 平台 | OAuth版本 | 回调地址 | 获取用户信息 |
|------|----------|---------|-------------|
| Google | OAuth 2.0 | /api/auth/callback/google | People API |
| GitHub | OAuth 2.0 | /api/auth/callback/github | User API |
| Apple | Sign in with Apple | /api/auth/callback/apple | JWT验证 |
| 微信 | 微信开放平台 | /api/auth/callback/wechat | 获取用户信息API |
| QQ | QQ互联 | /api/auth/callback/qq | get_user_info |
| 支付宝 | 支付宝授权 | /api/auth/callback/alipay | alipay.user.info.share |
