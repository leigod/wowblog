# 文档模块设计方案

## 一、概述

### 1.1 设计目标

在现有 Blog 系统基础上，增加独立的文档（Doc）模块，支持：
- 多文档书管理（DocBook）
- 树形文档结构
- 文档内搜索
- 子域名访问
- 与 Blog 共享用户、评论等基础功能

### 1.2 设计原则

- **保持现有架构不变**：Blog 模块完全独立，不做改动
- **Doc 模块独立**：数据表、API、组件完全独立
- **辅助功能共享**：用户、评论等通用功能复用
- **展示层独立**：Blog 和 Doc 各自独立的展示界面

## 三、数据库设计

### 3.1 文档书表（wb_docbook）

文档的集合，类似一本书的概念。

```sql
CREATE TABLE wb_docbook (
    id              INT PRIMARY KEY AUTO_INCREMENT COMMENT '文档书ID',
    name            VARCHAR(100) NOT NULL COMMENT '文档书名称',
    slug            VARCHAR(100) NOT NULL UNIQUE COMMENT 'URL标识',
    description     TEXT COMMENT '文档书描述',
    cover           VARCHAR(500) COMMENT '封面图URL',
    icon            VARCHAR(100) COMMENT '图标',

    -- 设置
    is_public       BOOLEAN DEFAULT TRUE COMMENT '是否公开',
    show_sidebar    BOOLEAN DEFAULT TRUE COMMENT '是否显示侧边栏',
    allow_comment   BOOLEAN DEFAULT TRUE COMMENT '是否允许评论',
    allow_search    BOOLEAN DEFAULT TRUE COMMENT '是否允许搜索',

    -- 主题
    theme           VARCHAR(50) DEFAULT 'default' COMMENT '主题样式',

    -- 首页文档（文档书首页显示的文档ID）
    home_doc_id     INT COMMENT '首页文档ID',

    -- 作者信息
    author_id       INT NOT NULL COMMENT '创建者ID',

    -- 排序
    sort_order      INT DEFAULT 0 COMMENT '排序',

    -- 时间
    createtime      BIGINT NOT NULL COMMENT '创建时间',
    updatetime      BIGINT NOT NULL COMMENT '更新时间',

    INDEX idx_slug (slug),
    INDEX idx_author (author_id),
    INDEX idx_sort (sort_order)
) COMMENT='文档书表';
```

### 3.2 文档表（wb_doc）

文档内容表，支持树形结构。

```sql
CREATE TABLE wb_doc (
    id              INT PRIMARY KEY AUTO_INCREMENT COMMENT '文档ID',
    docbook_id      INT NOT NULL COMMENT '所属文档书ID',

    -- 内容字段
    title           VARCHAR(255) NOT NULL COMMENT '文档标题',
    slug            VARCHAR(255) NOT NULL COMMENT 'URL标识（同文档书内唯一）',
    content         LONGTEXT COMMENT '文档内容（Markdown）',
    excerpt         TEXT COMMENT '摘要',

    -- 树形结构
    parent_id       INT DEFAULT 0 COMMENT '父文档ID，0为顶级',
    level           INT DEFAULT 1 COMMENT '层级深度',
    path            VARCHAR(500) COMMENT '树路径：0/1/2',
    sort_order      INT DEFAULT 0 COMMENT '同级排序',

    -- 状态
    status          ENUM('draft', 'published', 'hidden') DEFAULT 'draft' COMMENT '状态',

    -- 作者和时间
    author_id       INT NOT NULL COMMENT '作者ID',
    createtime      BIGINT NOT NULL COMMENT '创建时间',
    updatetime      BIGINT NOT NULL COMMENT '更新时间',
    pubtime         BIGINT COMMENT '发布时间',

    -- 统计
    view_count      INT DEFAULT 0 COMMENT '浏览次数',
    comment_count   INT DEFAULT 0 COMMENT '评论数',

    -- SEO
    seo_title       VARCHAR(255) COMMENT 'SEO标题',
    seo_keywords    VARCHAR(500) COMMENT 'SEO关键词',
    seo_description TEXT COMMENT 'SEO描述',

    UNIQUE KEY uk_docbook_slug (docbook_id, slug),
    INDEX idx_docbook (docbook_id),
    INDEX idx_parent (parent_id),
    INDEX idx_status (status),
    INDEX idx_sort (docbook_id, parent_id, sort_order),

    FOREIGN KEY (docbook_id) REFERENCES wb_docbook(id) ON DELETE CASCADE
) COMMENT='文档表';
```

## 四、后端 API 设计

### 4.1 API 路由结构

```
/api/docbook/*          - 文档书管理
├── GET    /list        - 获取文档书列表
├── GET    /:id         - 获取文档书详情
├── POST   /create      - 创建文档书
├── PUT    /:id         - 更新文档书
└── DELETE /:id         - 删除文档书

/api/doc/*              - 文档管理
├── GET    /list        - 获取文档列表（支持树形结构）
├── GET    /tree/:docbook_id - 获取文档树
├── GET    /:id         - 获取文档详情
├── GET    /by-slug/:docbook_slug/:doc_slug - 通过slug获取文档
├── POST   /create      - 创建文档
├── PUT    /:id         - 更新文档
├── DELETE /:id         - 删除文档
├── POST   /move        - 移动文档节点
└── GET    /:id/navigation - 获取文档导航（上一篇/下一篇）

/api/doc/search/*       - 文档搜索
├── GET    /global      - 全站搜索（blog + doc）
└── GET    /in-docbook  - 文档书内搜索
```

const route = useRoute()
const { docbookSlug, docSlug } = route.params

const docbook = ref(null)
const currentDoc = ref(null)
const navigation = ref({ prev: null, next: null })

### 7.3 前端处理

```typescript
// src/utils/subdomain.ts
export function checkDocSubdomain() {
  const hostname = window.location.hostname
  const docSubdomain = appStore.siteConfig.doc_subdomain

  if (docSubdomain && hostname.includes(docSubdomain.replace('.*', ''))) {
    return { isDocSubdomain: true }
  }
  return { isDocSubdomain: false }
}

// 路由守卫
router.beforeEach((to, from, next) => {
  const { isDocSubdomain } = checkDocSubdomain()

  if (isDocSubdomain && to.path === '/') {
    next('/docs')
  } else {
    next()
  }
})
```

## 十一、技术要点

### 11.1 树形结构存储

使用 **path 字段**（闭包表变体）：

```
path 格式：0/1/2/3
- 0 表示根节点
- 数字表示节点 ID
- 查询子节点：WHERE path LIKE '0/1/%'
- 查询层级：计算 / 的数量
```

### 11.2 Markdown 渲染

- 使用现有的 Markdown 解析器
- 增强功能：
  - 自动提取标题生成 TOC
  - 代码块语法高亮
  - 支持 Mermaid 图表（可选）

### 11.3 性能优化

- 文档树缓存（Redis）

- 大文档分页/分段加载
